# AFK Mode + Fuzzy Fallback Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship two opt-in fallback layers for unmatched prompts:
1. **AFK** — a dumb idle timer that, after `-afk-idle` (default 10 min) with no PTY output *and* no user input, injects `y\r` then `continue, Choose based on your project understanding.\r`, rearms, and loops until `Ctrl+Y a`.
2. **Fuzzy** — a generic detector that waits for the screen to be stable for `-fuzzy-stable` (default 3 s) and fires only when the last 15 lines contain a precise y/n vocabulary marker. Routes through the normal approval flow (respects `-delay`, memory dedup).

**Architecture:** Both layers live inside the existing proxy `Run` select loop. AFK uses one `*time.Timer` reset on any inputCh/outputCh activity. Fuzzy uses a separate `*time.Timer` armed whenever `hash(screen.Text())` changes; when it fires, the current text is re-checked against a narrow regexp, and a `MatchResult` is synthesised and funnelled through the same approval handling that regular detectors use. Config and CLI plumbing mirror the existing `delay`/`enabled` pattern (pointer-typed per-agent overrides, flag takes priority over config). Runtime toggles `Ctrl+Y a` and `Ctrl+Y f` are handled inline in the input path, sharing a single `prefixActive`-guarded block.

**Tech Stack:** Go 1.22+, `github.com/pelletier/go-toml/v2`, existing `internal/proxy` event-loop style. No new dependencies.

**Spec reference:** `docs/superpowers/specs/2026-04-22-afk-mode-design.md` (AFK sections + `Generic Fuzzy Fallback` section)

```go
// internal/detector/fuzzy.go
package detector

import (
	"regexp"
	"strings"
)

// fuzzyPattern matches precise y/n prompt markers. Intentionally narrow:
// bare "Yes" / "No" / "enter" / "confirm" are excluded because they
// appear too often in normal prose and code.
var fuzzyPattern = regexp.MustCompile(`(?i)\([yn]/[yn]\)|\[[yn]/[yn]\]|[yn]/[yn]\?|yes/no`)

// FuzzyMatch reports whether the last 15 lines of text contain a precise
// y/n prompt marker. The last-15-lines window keeps the detector focused
// on the currently visible prompt area.
func FuzzyMatch(text string) bool {
	lines := strings.Split(text, "\n")
	start := 0
	if len(lines) > 15 {
		start = len(lines) - 15
	}
	return fuzzyPattern.MatchString(strings.Join(lines[start:], "\n"))
}
```
